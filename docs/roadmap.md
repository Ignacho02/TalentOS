# Roadmap

## Objetivo

Recoger siguientes iteraciones y trabajo pendiente sin mezclarlo con resumenes de cambios ya cerrados.

## Prioridades sugeridas

### Persistencia y producto

- Conectar el estado principal a Supabase como fuente de verdad real.
- Separar mejor modo demo y modo produccion.
- Definir modelo multiusuario, roles y ownership de club.

### Calidad tecnica

- Asegurar instalacion reproducible para nuevas incorporaciones.
- Ampliar cobertura de tests mas alla de calculos de maduracion.
- Revisar validaciones y manejo de errores en importacion Excel y formularios.

### DataHub

- Completar calendario de carga de entrenamiento con flujo mas maduro.
- Definir que hacer con GPS, hoy bloqueado o sin implementar.
- Integrar de forma consistente color, identidad y escudo del club en toda la app.

### Modulos futuros

- Convertir `Community` en un modulo funcional con casos de uso reales.
- Convertir `Research` en un modulo funcional con contenido y flujos reales.

## Backlog operativo

- Revisar documentacion interna y mantener `docs/` como unica zona para notas de handoff.
- Definir si se anadira una carpeta `scripts/` para tareas de setup, importacion o mantenimiento.
- Volver a inicializar o integrar este directorio en un repositorio Git con convenciones limpias.
